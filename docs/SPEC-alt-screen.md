# SPEC — Alternate screen buffer + terminal title

> Запуск бармена в полноэкранном режиме: TUI занимает весь терминал, скрывая историю команд, а заголовок вкладки/окна меняется на «🍸 Бар». При выходе история и предыдущий заголовок восстанавливаются.

- [SPEC.md](./SPEC.md) · [BACKLOG.md](./BACKLOG.md)

## Проблема

Сейчас `src/index.tsx:7` рендерит `<App/>` через `render()` в основной экран терминала. История команд (предыдущие запуски, вывод shell) видна над интерфейсом бармена; Ink рисует «снизу» экрана. Это ломает иммерсивность и раскладку, рассчитанную на всю высоту терминала. Заголовок вкладки терминала остаётся тем, что поставил shell (обычно `tsx` или путь), — не брендует окно.

В Ink 5.2.1 в типе `RenderOptions` (`node_modules/ink/build/render.d.ts`) **нет** опции `fullscreen` — только `stdout/stdin/stderr/debug/exitOnCtrlC/patchConsole`. Альт-экран и заголовок нужно включать вручную.

## Дизайн

**Alternate screen buffer** — стандартный режим терминала (`vim`, `less`, `htop`). Управляется ANSI-кодами:

- `\x1b[?1049h` — войти (сохраняет текущий экран, открывает чистый)
- `\x1b[?1049l` — выйти (восстанавливает сохранённый экран)

Дополнительно при входе шлём `\x1b[2J\x1b[H` (clear + cursor home), чтобы некоторые терминалы не оставляли курсор посередине.

**Заголовок окна/вкладки** меняется OSC-последовательностью `\x1b]0;<title>\x07` (xterm title). Сохранение/восстановление предыдущего заголовка — через xterm title stack:

- `\x1b[22;0t` — сохранить текущий заголовок в стек
- `\x1b[23;0t` — достать заголовок из стека

Стек поддерживается xterm/iTerm2/Terminal.app/kitty/alacritty/Windows Terminal; если терминал не реагирует, заголовок просто останется нашим — приемлемый фолбэк.

### Безопасность выхода

Главная проблема альт-экрана — если процесс упадёт, не вернув код выхода, терминал останется в альт-экране (история «пропадёт»). Поэтому:

- `process.on("exit", ...)` — синхронный выход из альт-экрана. TTY-запись синхронна, поэтому сработает.
- `SIGINT`, `SIGTERM`, `SIGHUP` — явно вызывают `process.exit()` (что триггерит `exit`).
- `uncaughtException`, `unhandledRejection` — пишут диагностику в stderr, затем `process.exit(1)`.
- Все вызовы **идемпотентны** (флаг `active`).

### TTY-сторож

`process.stdout.isTTY` — `true` только в реальном терминале. В CI/vitest/pipe — `undefined`. Альт-экран включается только при `isTTY`, чтобы не ломать тесты и скрипты (`scripts/smoke.ts` и др.).

## Затронутые файлы

- `src/altScreen.ts` (новый) — `enterAltScreen(title?)` (вход + save title + set title), `exitAltScreen` (выход + restore title), `installAltScreenGuards`.
- `src/index.tsx` — `enterAltScreen("🍸 Бар")` и установка гардов перед `render()`.
- `src/shutdown.ts` — явный вызов `exitAltScreen` перед `process.exit(0)` (дубль с `exit`-хендлером, для надёжности).

## Тесты

Модуль тривиальный (запись байт в stdout + регистрация хендлеров). Юнит-тестов не требует — поведение проверяется вручную в реальном терминале. TTY-сторож гарантирует, что существующие тесты (`npm test`) не затронуты.
